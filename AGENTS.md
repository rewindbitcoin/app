# AGENTS.md

## Commands
- `npm run android` / `npm run ios` use `expo run:*`; this app requires a custom Expo dev build and will not run in Expo Go.
- `npm run test` runs `npm run lint -> npm run build:src -> npm run build:test -> npm run ensureTester -> jest`.
- Focused test loop: `npm run build:src && npm run build:test && npx jest test/vaults.unit.test.js`.
- Regtest suites (`test/basic-vault.test.ts`, `test/edge2edge.test.ts`) need Docker. `npm run ensureTester` pulls/starts `bitcoinerlab/tape` and exposes `8080/60401/3002/5123` by default; tests read `ESPLORA_PORT` and default to `3002`.
- `npm run lint` forces `ESLINT_USE_FLAT_CONFIG=false` and currently only lints `.ts` files under `src/` and `test/`; `.tsx` changes are not covered by the repo script.

## Structure
- `index.js` only registers `src/App.tsx`.
- `src/App.tsx` is the real app entrypoint: providers, navigation, splash handling, and lazy screen loading all live there.
- `init.ts` is imported by `src/App.tsx` and is the global RN boot/setup file: Android text workaround, Buffer/tcp/tls/URL polyfills, and startup logging. Put boot-time shims there instead of duplicating them.
- `src/app/` contains wallet/vault features, screens, contexts, and domain logic. `src/common/` is shared UI, theme, storage, and crypto infrastructure.
- `src/app/screens.ts` owns route constants/types and switches navigator implementation: native stack on mobile, JS stack on web.
- `src/app/contexts/WalletContext.tsx` switches explorer implementation by platform: native uses Electrum, web uses Esplora.
- `src/common/lib/storage.ts` defaults to `MMKV` on native and `IDB` on web; `SECURESTORE` requires an auth prompt and is reserved for biometric-sensitive data.

## Generated And Env
- `build:src` writes compiled runtime code to `dist/`; the tests import `../dist/src/...`, so stale `dist/` causes misleading failures.
- `build:test` emits `.js` next to `.ts` inside `test/`. Jest only matches `**/*.test.js`, so edit `.ts` sources and regenerate instead of hand-editing the checked-in JS/DTS files.
- `src/i18n-locales/init.ts` is autogenerated by `npm run i18n`; translation sources are `src/i18n-locales/en.js` and `src/i18n-locales/es.js`.
- `@env` is injected by Babel from `node_modules/@rewindbitcoin/env/services.env`, not from a repo-local `.env`.
- `src/app/lib/settings.ts` intentionally checks `VERSION` to force Metro/Babel to reload `@env`. If env values change, bump `VERSION` there and restart with `npx expo start --clear`.
- `npm install` runs `patch-package`. Native dependency fixes in `patches/` are required; read `patches/README.rewindbitcoin.md` before upgrading deps or debugging native build failures.
